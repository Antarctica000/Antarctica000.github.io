1.add方法与toString方法

ArrayList 类和简单数组都有其对应的 toString 实现，它们可以直接与 Arrays.toString() 配合使用。对于简单数组，Arrays.toString() 能够直接处理因为它被重载了多次以处理不同基本类型的数组（比如 int[], char[], double[] 等）以及对象数组（比如 String[], Integer[], Object[] 等）。
当你创建自己的动态数组类时，这个类是一个自定义的对象，不是数组类型。Arrays.toString() 方法不知道如何处理这个类的实例，因为它不是一个数组。它只能接受一个真正的数组作为参数。
如果你的动态数组类内部使用了一个原生数组来存储元素，你需要在类的 toString() 方法中调用 Arrays.toString() 并传入那个内部数组。
当你调用 System.out.println(instance);，Java 会自动调用你为你的动态数组类提供的 toString() 方法，而该方法会返回 Arrays.toString(array) 的结果，其中 array 是你的内部存储结构。这样就可以打印出动态数组的内容了

2.size为0时第一次添加index为0怎么办？怎么处理这个逻辑？看看源码别人是怎么写的。

size的两个含义：元素个数和下次应存的位置。

状态模式允许一个对象在其内部状态改变时改变其行为。这似乎是通过改变其所属的类一样。对于游戏，我们可以为每个场景定义一个状态类，并在游戏上下文中切换状态。

命令模式

命令模式是另一个行为设计模式，它将一个请求封装成一个对象，从而让你使用不同的请求、队列或日志请求来参数化其他对象。它同时也支持可撤销的操作。在游戏中，命令模式可以用来表示玩家的行为，如攻击、移动等。

1. 统计方形（普及-）2/28

题目描述
有一个 $n \times m$ 方格的棋盘，求其方格包含多少正方形、长方形（不包含正方形）。
输入格式
一行，两个正整数 $n,m$（$n \leq 5000,m \leq 5000$）。

并不完全暴力，使用了排列组合的数学公式进行简化。其中WA了两次，都是因为忽略了m与n可能数据过大而造成的溢出。所以最好使用long的数据类型。
如果用暴力枚举来做的话，可以通过选取矩形的左上角和右下角的点来枚举所有的矩形。时间复杂度甚至达到O(n4)，好处是思想简单。

2. 烤鸡（普及-）2/28

题目描述：
猪猪 Hanke 特别喜欢吃烤鸡（本是同畜牲，相煎何太急！）Hanke 吃鸡很特别，为什么特别呢？因为他有10种配料（芥末、孜然等），每种配料可以放1到3克，任意烤鸡的美味程度为所有配料质量之和。现在，Hanke 想要知道，如果给你一个美味程度 n ，请输出这10种配料的所有搭配方案。

怀疑暴力，理解暴力，欣赏暴力，成为暴力
可恶的是题目要求先输出方案再输出字典序的排列。这样我就不得不使用两次一摸一样的循环，除非想个办法存储一大堆字符串。好在pow(3,10)=59049，for多一点也问题不大。

正解是DFS。可扩展性、简洁性都会更强。不过就这题而言使用for当然是更快速更好的选择

这是递归函数中的一个常见模式：
选择：将当前选项添加到路径中（combination.add(i);）。
探索：继续递归探索下一个选项（dfs(depth + 1, targetSum - i, combination);）。
撤销选择（回溯）：将当前选项从路径中移除（combination.remove(combination.size() - 1);）。
为什么需要回溯呢？让我们以一个简化的例子来说明。假设我们只有两种配料A和B，每种配料可以选择1到3克，我们希望找到总量为4克的所有配料组合。
一开始，我们选择了A的1克，然后我们需要探索B的所有可能性：

  1. A=1, B=1（不合适，因为总和为2）
  2. A=1, B=2（不合适，因为总和为3）
  3. A=1, B=3（合适，因为总和为4）

完成A为1克的所有可能性后，我们需要回溯，也就是撤销A为1克的选择，然后探索A为2克的所有可能性：

  1. A=2, B=1（不合适，因为总和为3）
  2. A=2, B=2（合适，因为总和为4）

同理，我们也需要探索A为3克的情况。在每次探索完成后，我们都需要回溯到上一个状态，以便不遗漏任何可能的组合。如果不进行回溯，我们就不能穷举所有可能的组合，因为我们会遗失掉一些选择路径。

4. 选数（普及-）2/29

在深度优先搜索（DFS）中，“回溯”是指递归调用结束后，程序的执行状态返回到递归调用之前的状态。在您的代码中，这种行为已经隐式地发生了。每次递归调用 dfs 方法时，当前状态（包括当前的和 sum 和起始索引 start）都被压入调用栈。当递归调用完成时，程序的执行状态自然就“回溯”到了这些变量的之前状态。

具体来说，在您的代码中，dfs 方法中的每一次递归调用都创建了一个新的执行上下文，其中包括了对应的 k、sum 和 start 的副本。当一个递归调用完成并返回时：

  • 变量 k 的值恢复到调用前的状态，因为 k-1 并没有改变当前递归层次中 k 的值。
  • 变量 sum 的值也恢复到调用前的状态，因为 sum + a[i] 是传递给下一个递归调用的，当前层次的 sum 并没有被改变。
  • 变量 start 的值同样恢复到了调用前的状态，因为每次递归都是传递了 i+1，而不是修改 start 本身。

这个过程实际上已经包含了回溯算法的本质，即探索所有可能的解决方案，并在达到一定条件（如这里的 k == 0）时记录结果，然后返回到上一步，尝试其他可能的路径。

所以，你的代码中虽然没有显式的“撤销”操作，但通过递归调用和局部变量的作用域机制，实现了回溯搜索的效果。

在某些其他类型的 DFS 问题中，你可能会看到显式的回溯，例如，在递归之前选择一个元素（如放入一个数组），在递归之后撤销这个选择（如从数组中移除）。但在这个特定的问题中，由于你是计算和的值并检查它是否为素数，所以并不需要显式地撤销选择，因为每次递归调用都是基于不同的参数值。
